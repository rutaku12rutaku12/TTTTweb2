[ 웹 크롤링 ]
1. 정의 : 인터넷 웹 페이지에서 원하는 데이터(html)를 수집하는 기술

2. 웹 페이지 구동 원리 2가지
    1) 정적 페이지 : HTML 소스에 데이터가 있는 경우 , 즉] 게시물의 정보가 포함된 HTML
        * Jsoup 라이브러리 사용
    2) 동적 페이지 : JS가 HTML 소스에 데이터를 넣어주는 경우 , 즉] 게시물의 정보가 없는 HTML
        * Selenium 라이브러리 사용

3. 목적
    1) 데이터 수집 : 뉴스 , 상품 , 날씨 , 위치 등등
    2) 데이터 분석 : 수집한 자료를 저장 -> 통계 -> 학습(AI)
    3) 업무 효율 : 자동으로 특정 버튼 클릭, 자동으로 아이디 비밀번호 입력 등등
4. robots.txt : 웹사이트 루트 경로에 있는 자동화(크롤링) 접근 규칙 파일
    1) 웹페이지주소/robots.txt
       https://www.daum.net/robots.txt
    2) 접근 권한
        Allow : 접근 허용 한다 뜻
        Disallow : 접근 금지
    * 즉) 법적 강제성은 없지만, 웹크롤링 한번이 REST 요청 1번이라서 데이터베이스 작동 -> 과부하
[ Jsoup ]
    1. 그레이들 설치 , https://mvnrepository.com/
        1) jsoup
        2) Jsoup Java HTML Parser
        3) implementation 'org.jsoup:jsoup:1.21.2'

    2. 주요 메소드
        1) 크롤링할 웹주소의 HTML를 자바의 Document 객체로 가져오기
            Document document = Jsoup.connect( 크롤링할웹주소 ).get()

        2) 특정한 마크업 가져오기 , 1개 일때 Element / N개 일때 : Elements
            document.select( "css 선택자")
            Element element = document.selectFirst( "css 선택자" )
                예] <a href=""> 부평역 </a>
            Elements elements = document.select( "css 선택자" )
                예] [ <a href=""> 부평역 </a> , <a href=""> 부천역 </a> ]

        3) 마크업(요소)의 데이터 가져오기
            String text = element.text();
                예] <a href=""> 부평역 </a> --> 부평역
        4) 마크업(요소)의 속성값 가져오기
            String attr = element.attr("속성명");
                예) <img src="localhost:8080" /> --> localhost:8080
