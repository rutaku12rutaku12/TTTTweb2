[ 빌더패턴 ]
1. 정의 : lombok 라이브러리 에서 제공하는 빌더 패턴
2. 목적 : 복잡한 객체 생성을 간단하고 직관적으로 표현
3. 장점 :
    1) 매개변수가 많을때 유용 (순서규칙 없음)
    2) 필요한 매개변수만 선택적으로 초기화
4. 주요메소드
    1) builder() : 빌더 객체 시작점 static
    2) .속성명(초기값) : 지정한 속성명에 초기값 대입
    3) build() : 빌더 객체 끝지점
5. 사용법
    1) 빌더패턴을 사용할 클래스위에 빌더패턴 관련 어노테이션 주입
        @Builder
        @NoArgsConstructor @AllArgsConstructor
    2) 빌더패턴 사용
        [순수자바] 클래스명 변수명 = new 생성자명( 값1 , 값2 );
                        vs
        [빌더패턴] 클래스명 변수명 = 클래스명.builder.속성명1(값1).속성명2(값2).build

* static 이란? 정적변수/메소드, 즉 객체없이 사용되는 변수/메소드
---> 객체명.메소드 [x] , 클래스명.메소드 [o]
---> 관례적으로 객체는 변수에 저장되므로 소문자 , 클래스는 대문자 로 시작
